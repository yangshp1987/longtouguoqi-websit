# 🎬 视频教学脚本

> 📅 场景：视频会议屏幕共享，技术工程师演示，甲方 1-2 位同事跟学  
> ⏱️ 时长：约 50-60 分钟  
> 🎯 目标：教会甲方独立完成"改文字、加新闻、换图片"三件事  
> 📹 要求：全程录屏保存，方便甲方后续随时回看

---

## 📋 开场准备（会前 10 分钟）

### 工程师端
- [ ] 电脑 CodeBuddy IDE 已打开，项目 `longtouguoqi-website` 已克隆好
- [ ] 浏览器已登录：腾讯云控制台、GitHub 仓库、网站 www.longtouguoqi.com.cn
- [ ] 屏幕分辨率调到 1920×1080，字体调大便于甲方观看
- [ ] 开启屏幕录制（OBS / 腾讯会议录屏）

### 甲方端（提前让他们准备）
- [ ] CodeBuddy IDE 已安装并登录
- [ ] GitHub 账号已登录
- [ ] 有一张"想换上去的新图片"（随便什么图都行，用于练习）
- [ ] 准备一段"想加到新闻页的文字"（一两句话即可）

## 🎞️ 第三幕：第一个练习 — 改文字（10 分钟）

### 演示任务

> "我们来把首页的电话号码 13072913676 改成测试号码 13800138000，  
> 改完看到网站更新，再改回来。"

### 逐步操作

**Step 1**：在 AI 聊天窗口输入：

```
请把首页的电话号码 13072913676 改成 13800138000，全文所有地方都要改
```

**Step 2**：AI 会自动找到 `index.html` 里所有电话号码的位置并修改，  
在代码编辑器里可以看到**绿色（新增）和红色（删除）**的变化，  
点击"**接受更改**"。

**Step 3**：左下角 Git 面板

```
- 看到文件列表里 "index.html" 有个 M 标记（Modified 已修改）
- 在"消息框"里输入：把电话改为测试号码
- 点击 ✓ Commit（提交）
- 点击 ⬆ Push（推送）
```

**Step 4**：切换到浏览器，打开 GitHub 仓库页面

```
- 能看到最新提交记录"把电话改为测试号码"
- 点击 "Actions" 标签，能看到 EdgeOne 正在部署（转圈）
- 等 1-2 分钟，变成绿色 ✓ 表示完成
```

**Step 5**：打开 www.longtouguoqi.com.cn，按 `Ctrl+F5` 强制刷新

```
✅ 看到首页电话号码已经变成 13800138000
```

**Step 6**：练习改回来

```
在 AI 聊天窗口输入：
"把电话号码改回 13072913676"

→ 重复 Step 2-5 的流程
→ 再次验证网站恢复正常
```

## 🎞️ 第四幕：第二个练习 — 加新闻（15 分钟）

### 演示任务

> "我们来给新闻页加一条新闻。"

### 让甲方先准备文字内容

```
标题：龙头国漆亮相 2026 西安文创展
日期：2026-04-19
内容：近日，龙头国漆作为陕西省级非物质文化遗产代表性传承人企业，
     亮相 2026 西安文创博览会。现场展示了公司精心打造的《闭花羞月》系列
     漆器作品，吸引了众多参观者驻足观赏。
来源：本站原创
```

### AI 指令（让甲方自己输入）

```
在新闻页的最顶部加一条新闻，标题是"龙头国漆亮相 2026 西安文创展"，
日期是 2026-04-19，来源是"本站原创"，
内容是：近日，龙头国漆作为陕西省级非物质文化遗产代表性传承人企业，
亮相 2026 西安文创博览会。现场展示了公司精心打造的《闭花羞月》系列
漆器作品，吸引了众多参观者驻足观赏。
```

### AI 会自动

- 打开 `news.html`
- 在新闻列表最上方插入一个新卡片（模仿已有的新闻卡片结构）
- 保持样式一致

### 甲方验收检查点

- [ ] 在本地预览（CodeBuddy 有预览按钮）能看到新闻出现在新闻页顶部
- [ ] 样式和其他新闻一致
- [ ] 日期、标题、内容、来源都对
- [ ] Commit 消息写："2026-04-19 新增一条新闻"
- [ ] Push 之后等 1-2 分钟，线上网站显示新闻 ✅

### ⭐ 重点强调

> "每加一条新闻，Commit 消息都写清楚日期+内容，这样将来查历史很容易。"

## 🎞️ 第五幕：第三个练习 — 换图片（10 分钟）

### 演示任务

> "我们把首页 Hero 区域的那张图换成新的图。"

### 三步完成

**Step 1**：准备图片

```
- 让甲方从桌面拖一张 JPG 图片到 CodeBuddy 的 images/ 文件夹
- 重命名为 hero-new.jpg（英文名、没有空格）
- 尺寸建议 1200×900 以上，大小 < 500KB（太大会变慢）
```

**Step 2**：AI 指令

```
请把首页 Hero 区域的图片从 hero.jpg 换成 hero-new.jpg
```

**Step 3**：Commit + Push + 等 1-2 分钟刷新验证

### ⚠️ 图片常见坑（一定要讲）

```
❌ 坑 1：文件名带中文 → 可能显示不出来
     → 永远用英文 + 数字 + 短横线，例如 product-01.jpg

❌ 坑 2：图片太大（>1MB）→ 网页加载变慢
     → 用手机自带的压缩功能，或用 tinypng.com 压缩后再上传

❌ 坑 3：删掉了旧图片，发现别的地方还在用它 → 其他页面报错
     → 删图前，先问 AI："这张图还有别的地方在用吗？"
```

## 🎞️ 第六幕：应急救援 — 改坏了怎么办（10 分钟）

### 场景模拟

> "假设你改完网站，刷新后发现网站全白了！怎么办？"

### 一键回滚流程（⭐必练）

**Step 1**：打开浏览器访问

```
https://github.com/<甲方账号>/longtouguoqi-website/commits/main
```

**Step 2**：找到**上一个好的版本**

```
每一条提交记录右边都有一个 "Revert" 按钮
点击"Revert"（回滚）
```

**Step 3**：GitHub 会让你确认

```
点击"Create pull request"
点击"Merge pull request"
点击"Confirm merge"

→ 1-2 分钟后，网站自动恢复到上一个版本 ✅
```

### ⭐ 练习：让甲方亲自操作一次

> "来，你们自己在刚才改的新闻基础上回滚一次，  
> 确认网站上那条测试新闻消失。"
