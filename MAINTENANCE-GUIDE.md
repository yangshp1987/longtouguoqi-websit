## 二、推荐方案：三层交付 + AI 辅助维护

整个方案分**三层**，每一层独立承担不同的职责，三层加起来刚好覆盖"资产所有权 → 日常维护 → 紧急救援"三种场景。

```
┌─────────────────────────────────────────────────────┐
│  Layer 1: 资产所有权层（一次性）                     │
│  把域名、服务器、代码仓库全部转到甲方自己的账号下     │
├─────────────────────────────────────────────────────┤
│  Layer 2: 日常维护层（AI + GitHub）                  │
│  CodeBuddy IDE + GitHub 仓库 + EdgeOne 自动部署      │
├─────────────────────────────────────────────────────┤
│  Layer 3: 应急救援层                                 │
│  一键回滚 + 技术顾问服务协议                         │
└─────────────────────────────────────────────────────┘
```

## 三、Layer 1：资产所有权转移（一次性，约 2-3 小时）

### 3.1 让甲方注册自己的账号（清单）

| 账号 | 用途 | 注册地址 | 备注 |
|------|------|---------|------|
| **腾讯云账号** | 域名/备案/部署 | cloud.tencent.com | 用公司**对公邮箱**+**法人手机号** |
| **GitHub 账号** | 代码仓库 | github.com | 免费账号即可 |

### 3.2 域名转移

```
技术工程师账户 → 甲方腾讯云账户
路径：DNSPod 控制台 → 域名列表 → 长按域名 → 账号间迁移

⚠️ 需要甲方账号的 UIN（用户ID），扫码或邮件确认即可
⚠️ ICP 备案主体已经是客户公司，域名转移不影响备案
```

### 3.3 EdgeOne Pages 项目转移

```
方式 A（推荐）：团队协作
  - 甲方在自己腾讯云账户创建同名 EdgeOne Pages 项目
  - GitHub 仓库授权给甲方
  - 重新部署一次（自动生成新 CNAME）
  - DNSPod 里 www 的 CNAME 改成新值

方式 B：项目转让
  - 腾讯云支持项目在账号间转让
  - 联系腾讯云客服走转让流程
```

### 3.4 源代码交付

```
工程师操作：
1. 在 GitHub 甲方账号下创建仓库 longtouguoqi-website
2. 把本地 /官网/ 目录推上去
3. 仓库设为 Private（私有）
4. 把工程师本人加为 Collaborator（协作者，可随时移除）
```

## 四、Layer 2：日常维护层 — 小白也能上手

### 4.1 维护工具选型

| 工具 | 用途 | 学习成本 |
|------|------|---------|
| **CodeBuddy IDE** (VS Code + AI 插件) | 改网站文字、加新闻、换图片 | ⭐⭐（3个按钮） |
| **GitHub** | 代码仓库（自动备份+历史版本） | ⭐（只用网页界面） |
| **EdgeOne Pages** | 自动部署（改完代码自动上线） | ⭐（配置一次就不用管） |

### 4.2 真正的傻瓜式工作流

```
┌────────────────────────────────────────────────────┐
│                                                    │
│  甲方想改网站（加新闻 / 改电话 / 换图片）         │
│                   ↓                                │
│  打开 CodeBuddy IDE                                │
│                   ↓                                │
│  用中文说："在新闻页加一条 XXX 新闻"               │
│                   ↓                                │
│  AI 自动改代码（在本地预览）                       │
│                   ↓                                │
│  点"保存并提交"按钮                                │
│                   ↓                                │
│  GitHub 收到代码                                   │
│                   ↓                                │
│  EdgeOne Pages 自动构建并上线（1-2分钟）           │
│                   ↓                                │
│  ✅ 网站已更新                                     │
│                                                    │
└────────────────────────────────────────────────────┘
```

### 4.3 GitHub → EdgeOne Pages 自动部署配置（一次性）

工程师在交付时帮甲方配置好：

```
1. 进入 EdgeOne Pages 项目控制台
2. "项目设置" → "源" → 选择"从 GitHub 仓库部署"
3. 授权绑定 longtouguoqi-website 仓库
4. 分支选 main，构建命令留空，发布目录填 .
5. 保存

→ 从此以后，只要甲方改代码推到 GitHub，EdgeOne 自动部署
→ 甲方完全不用碰服务器
```

### 4.4 甲方真正要学的 3 件事

#### ① 改一条文字
```
打开 CodeBuddy IDE → 对 AI 说：
"把首页的电话号码改成 123-4567-8900"
→ AI 自动改 → 点"提交" → 完成
```

#### ② 加一条新闻
```
打开 CodeBuddy IDE → 对 AI 说：
"在新闻页加一条今天的新闻，标题是 XXX，内容是 YYY"
→ AI 自动改 → 点"提交" → 完成
```

#### ③ 换一张图片
```
把新图片拖进 images/ 文件夹 → 对 AI 说：
"把首页的 hero 图换成我刚放进去的那张 new-hero.jpg"
→ AI 自动改 → 点"提交" → 完成
```

> ❌ **甲方不需要学**：HTML、CSS、JavaScript、命令行、Git 命令、DNS 配置、服务器

### 4.5 给甲方的"禁忌清单"

```
⚠️ 除非和技术顾问确认，否则不要：
  1. 删除 *.html 文件
  2. 删除 images/ 整个文件夹
  3. 修改 sitemap.xml 之外的 XML 文件
  4. 修改 robots.txt
  5. 改域名相关配置
```

