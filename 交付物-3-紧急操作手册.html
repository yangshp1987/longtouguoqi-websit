三 · 三大高频救援场景 03
场景 1 网站打不开
1
先用手机 4G 流量打开 www.longtouguoqi.com.cn，确认不是自己 WiFi 问题
2
如果真打不开，打开 GitHub 仓库 commits/main 页面，看最近 30 分钟内有没有自己的提交
3
如果 30 分钟内有过提交 → 大概率是改坏了，立刻执行"一键回滚"（见场景 2）
4
如果不是改动造成的 → 立刻联系技术顾问
场景 2 一键回滚 · 改坏了怎么救
1
打开网页 github.com/【账号】/longtouguoqi-website/commits/main
2
找到"改坏之前"的那条记录（看日期）
3
点击那条记录右边的 Revert 按钮
4
依次点击：Create pull request → Merge pull request → Confirm merge
5
等 1-2 分钟，用 Ctrl+F5 强制刷新网站 → 网站恢复 ✓
场景 3 域名续费 · 最容易忘记的事
1
每年 1 月 1 日、7 月 1 日 各检查一次
2
登录腾讯云控制台 → 域名管理 → 查看到期时间
3
到期前 30 天在腾讯云续费（几十元一年）
4
SSL 证书 → 自动续期，不用管

四 · 日常维护速查 04
想做什么打开 CodeBuddy 对 AI 说
改电话号码把首页电话 xxx 改成 xxx，所有地方都要改
改公司地址把联系页的地址改成 xxxx
加一条新闻在新闻页顶部加一条：标题 xxx，日期 xxx，内容 xxx
换首页大图（先拖新图到 images/）把首页 hero 图换成 xxx.jpg
改产品价格把产品页 xxx 的价格从 xxx 改成 xxx
加一个店铺链接在首页渠道区加一个 xxx 链接，地址是 xxx
改完别忘记 3 步：
① 点"接受更改" → ② 左下角 Git 写提交消息（今天日期+改了啥） → ③ 点 Commit ✓ 和 Push ⬆
⛔ 绝对不要做
  • 删除任何 .html 结尾的文件
  • 删除整个 images/ 文件夹
  • 修改 robots.txt、sitemap.xml
  • 修改带 "schema.org" 的代码块
  • 在腾讯云后台点击"删除项目"
  • 告诉陌生人任何账号密码
  • 周五下班前做大改动
✓ 推荐养成的习惯
  • 每周：打开网站看一眼
  • 每周：看 GitHub 有无异常提交
  • 每月：发布 1-2 条新闻
  • 每月：检查店铺链接是否有效
  • 每月：备份 images/ 到公司电脑
  • 每半年：检查域名到期时间
  • 每半年：检查 ICP 备案信息
五 · 账号安全守则 05
账号安全建议
GitHub开启 两步验证（短信 / 验证码 APP）
腾讯云开启 两步验证 + 设置 操作保护
CodeBuddy设置强密码（≥ 12 位，含大小写+数字+符号）
企业邮箱绝对不能丢——所有账号找回都靠它
📌 密码管理建议：
3 个账号密码写在一张纸上，锁在保险柜里；
或用 1Password / 微信密码管理器 统一管理。
